What Each Does Better

Neither system is universally superior. Each has areas where it leads, and substantial common ground.

Sinapsis Excels At

#1
Broader Ecosystem
Skill router, operator state, passive rules. Not just learning, but a full skill management system with on-demand loading.
#2
Operator State
Strategic decisions persist across all projects. Identity, stack decisions, retired tech warnings — loaded every session.
#3
Passive Rules Engine
6 deterministic rules that fire on regex match. Separate from confidence-based instincts — guaranteed to trigger.
#4
Portability Commands
/backup, /restore, /cleanup as first-class commands with cloud sync documentation for all major providers.
#5
Token Efficiency
6,915 tokens/session for 3 global skills. Instinct injection capped at 1,500 chars per tool-use event.

Cortex Excels At

#1
Testing Depth
159 tests in 12 suites. ShellCheck + flake8 lint in CI. Dedicated PowerShell test suite for Windows coverage.
#2
Granular Confidence
Continuous 0.00–0.95 allows fine-tuned calibration. Rejection rate tracking for feedback-aware learning.
#3
Knowledge Timeline
Append-only log with 11 event types. Full audit trail of every knowledge creation, promotion, decay, and deletion.
#4
Law Distillation
Instincts mature into one-liner laws injected every session. An extra tier of crystallized, battle-tested wisdom.
#5
Agent Architecture
3 built-in agents: Observer (Opus), Reviewer (Sonnet x3), Planner (Sonnet). Agent evolution from detected patterns.
#6
Security Depth
12 scrubbing patterns, HOME/CWD validation, YAML import validation, uninstall safety guard, flock/perl atomic write fallback.

Equivalent / Shared Ground

Dream Cycle (5 modules, same concept)
Observation pipeline (async JSONL)
Context bridge (context.md per project)
Error-fix detection
Domain pre-filtering
Multi-project EOD
Auto-promote with multi-session gate
Downvote / feedback (both recent)
CI/CD on 3 operating systems
Pre-push hooks for test safety

Improvement Proposals

Each system can learn from the other. These are concrete, actionable proposals ranked by effort-to-value ratio.

For Sinapsis

Ideas borrowed from Cortex
1
Lint Integration in CI
Add ShellCheck + flake8 to GitHub Actions. Low effort, high value. Catches bugs before they reach hooks.
2
Knowledge Timeline
Append-only event log tracking instinct lifecycle: creation, promotion, decay, merge, deletion. Full audit trail.
3
Instinct Import Validation
Block malicious or malformed content on /instinct-import. Validate structure, cap size, reject injection attempts.
4
Uninstall Command
Safe uninstall with guard rails. Confirm before deletion, backup before removal, clean hooks from settings.json.
5
Agent Architecture for /analyze-session
Use Opus 1M for deeper semantic analysis instead of a single Claude call. Multi-agent review for higher quality instincts.

For Cortex

Ideas borrowed from Sinapsis
1
Operator State
Cross-project strategic decisions. Stack choices, retired tech warnings, brand context — loaded once, applied everywhere.
2
Passive Rules Engine
Deterministic guardrails separate from confidence-based instincts. Regex triggers, guaranteed fire, no decay.
3
Skill Router / On-Demand Loading
Reduce always-on token cost. Load skills only when needed, from a dormant library. Tag-based matching.
4
/cleanup Command
Dedicated homunculus hygiene: remove orphaned files, consolidate duplicates, prune stale observations.
5
Token Budget Cap on Injection
Prevent runaway instinct injection. Explicit char limit per tool-use event to control token overhead predictably.
