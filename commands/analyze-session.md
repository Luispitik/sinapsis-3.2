# /analyze-session -- Session Analysis

> Review proposals generated by the session-learner (error-resolution patterns)
> and promote them to confirmed instincts. This is the manual validation step
> in the v4.1 closed learning pipeline.

---

## Trigger

Run with `/analyze-session`, "analyze session", or "review proposals".

---

## Process

### Step 1: Load Proposals

Read `~/.claude/skills/_instinct-proposals.json` and gather:
- Proposals with status `pending`
- Session date and proposal count
- Evidence (which tool, what error, how it was resolved)

Also read `~/.claude/skills/_instincts-index.json` to check for:
- Already confirmed instincts (avoid duplicates)
- Current instinct count by domain

### Step 2: Present Findings

```
SESSION ANALYSIS

  Proposals file: _instinct-proposals.json
  Session date:   2026-03-31
  Pending:        3 proposals

  ============================================
  PROPOSALS TO REVIEW
  ============================================

  1. "fix-edit" (error_resolution)
     Evidence: Edit error resolved — possible gotcha to document
     Session: 2026-03-31
     Domain:  code-quality
     [C] Confirm as instinct  [X] Discard

  2. "fix-bash" (error_resolution)
     Evidence: Bash error resolved — possible gotcha to document
     Session: 2026-03-31
     Domain:  tooling
     [C] Confirm as instinct  [X] Discard

  [... more proposals ...]

  Enter choices (e.g., "1C 2X"): _
```

### Step 3: For Each Confirmed Proposal

Ask the user to refine before adding:

```
Confirming "fix-edit":

  What should Claude do differently next time?
  (Write the inject message — what Claude should remember)

  Example: "When editing files, always verify the file exists first
  with Read before attempting Edit."

  Trigger pattern (regex, matches tool+input):
  Example: "Edit|Write" or "\.tsx|\.ts"

  Domain: [code-quality / tooling / git / security / testing / custom]
```

### Step 4: Execute

For each confirmed proposal:
- Add to `_instincts-index.json` with:
  - `level`: "confirmed"
  - `trigger_pattern`: user-provided regex
  - `inject`: user-provided message
  - `domain`: user-selected
  - `origin`: "session-learner"
  - `added`: today's date
- Remove from `_instinct-proposals.json` (mark as processed)

For discarded proposals:
- Mark as `status: "discarded"` in proposals file

### Step 5: Summary

```
SESSION ANALYSIS COMPLETE

  Reviewed:    3 proposals
  Confirmed:   2 → added to _instincts-index.json
  Discarded:   1

  Active instincts: 5 (was 3)
  Domains covered:  code-quality, security, git, tooling

  These instincts will be injected automatically when their
  trigger pattern matches future tool use.
```

---

## Edge Cases

- **No proposals file**: Show "No proposals found. The session-learner generates proposals when it detects error→resolution patterns. Run a few sessions first."
- **All proposals already processed**: Show "All proposals have been reviewed. New proposals appear after sessions with error→resolution patterns."
- **Empty proposals array**: Same as no proposals.
- **Duplicate domain**: Warn user — "You already have a confirmed instinct in domain 'code-quality'. The instinct-activator uses domain dedup (one per domain, max 3). The higher-level instinct will take priority."
